## 2. 新增技术选型

| 职责 | 库 | 理由 |
|------|----|------|
| 音频播放 | `pygame.mixer` | 原生支持 MP3/WAV/OGG，提供 pause / unpause / set_pos，无需 ffmpeg 依赖 |

## 4. F1：音频播放器模块

### 4.1 AudioPlayer 设计

**核心状态机：**

```
IDLE ──load()──► READY
READY ──play()──► PLAYING ──pause()──► PAUSED
PAUSED ──resume()──► PLAYING
PLAYING/PAUSED ──stop()──► READY
```

**播放位置计算（关键）：**

`pygame.mixer.music.get_pos()` 存在两个问题：
1. seek 后归零，无法直接累加得到绝对位置
2. 与系统时钟存在轻微漂移

因此改用 wall-clock 方案：

```python
# play(start_pos) 时记录：
self._wall_start = time.time()    # 系统时间戳
self._pos_start  = start_pos      # 本次播放的起始位置（秒）

# get_position() 实现：
def get_position(self) -> float:
    if self._state == "PAUSED":
        return self._paused_pos
    if self._state == "PLAYING":
        elapsed = time.time() - self._wall_start
        return min(self._pos_start + elapsed, self._duration)
    return 0.0
```

**seek 实现：**

```python
def seek(self, pos: float) -> None:
    pos = max(0.0, min(pos, self._duration))
    if self._state == "PLAYING":
        pygame.mixer.music.play(0, start=pos)   # 从新位置重新播放
        self._wall_start = time.time()
        self._pos_start  = pos
    elif self._state == "PAUSED":
        self._paused_pos = pos                   # 暂停态只记录位置，不实际 play
```

**完整接口定义：**

```python
class AudioPlayer:
    def load(self, path: str) -> float:
        """加载文件，返回总时长（秒）"""

    def play(self, start_pos: float = 0.0) -> None:
        """从 start_pos 秒处开始播放"""

    def pause(self) -> None:
        """暂停，保存当前位置"""

    def resume(self) -> None:
        """从暂停位置继续播放"""

    def seek(self, pos: float) -> None:
        """跳转到指定秒数"""

    def stop(self) -> None:
        """停止并复位"""

    def get_position(self) -> float:
        """返回当前播放位置（秒），线程安全"""

    @property
    def duration(self) -> float: ...

    @property
    def state(self) -> str:
        """'IDLE' | 'READY' | 'PLAYING' | 'PAUSED'"""
```

### 4.2 GUI 播放控制区（AudioPlayerBar）

**布局：**

```
┌─────────────────────────────────────────────────────────┐
│  [◀◀]  [▶ 播放 / ⏸ 暂停]  [■ 停止]                      │
│  0:00 ├────────●──────────────────────┤ 3:42  [01:23]  │
└─────────────────────────────────────────────────────────┘
```

- `◀◀` 按钮：跳回开头（seek(0)）
- `▶/⏸` 按钮：切换播放 / 暂停状态，文字联动
- `■` 按钮：停止
- 进度条（CTkSlider）：range = [0, duration]，拖动时调用 seek()
- 时间标签：左侧显示总时长，右侧实时显示当前位置

**进度条同步策略：**

使用 `after(100, _poll)` 每 100ms 轮询一次播放位置，更新 Slider 和时间标签。

需要区分"用户拖动"和"程序更新"，防止反向干扰：

```python
self._user_dragging = False   # 鼠标按下时置 True，释放后置 False

def _poll(self):
    if not self._user_dragging:
        pos = self._player.get_position()
        self._progress_var.set(pos)
        self._lbl_cur.configure(text=_fmt_time(pos))
    self.after(100, self._poll)

# Slider 绑定：
slider.bind("<ButtonPress-1>",   lambda e: setattr(self, '_user_dragging', True))
slider.bind("<ButtonRelease-1>", self._on_seek_release)

def _on_seek_release(self, event):
    self._user_dragging = False
    self._player.seek(self._progress_var.get())
```

### 5.2 音符渲染算法

**核心变量：**

```
fall_speed      : 像素/秒，由 Speed Slider 控制（范围 100~800，默认 300）
canvas_height   : Canvas 高度（像素）
judgement_y     : 判定线 Y 坐标 = canvas_height * 0.85
visible_window  : 屏幕内可见的时间窗口（秒）= canvas_height / fall_speed
```

**每帧渲染逻辑（约 60fps，after(16, _tick)）：**

```python
def _tick(self):
    t_now = self._player.get_position()
    self._canvas.delete("note")   # 只删除音符层，保留静态背景

    for event in self._chart["chartData"]:
        t_note = event["time"]
        dt = t_note - t_now       # 距离判定的剩余秒数（负 = 已过）

        # 只渲染可见区间内的音符，稍微留一点余量
        if dt < -0.2 or dt > self._visible_window + 0.1:
            continue

        y_center = self._judgement_y - dt * self._fall_speed

        for lane_idx, track_val in enumerate(event["tracks"]):
            if track_val == "none":
                continue
            x1, x2 = self._lane_bounds(lane_idx)
            color   = LANE_COLORS[lane_idx]

            if track_val == "single":
                self._draw_tap(x1, x2, y_center, color)
            elif track_val.startswith("hold_"):
                duration = float(track_val[5:])
                hold_h   = duration * self._fall_speed
                self._draw_hold(x1, x2, y_center, hold_h, color)

    self._after_id = self._canvas.after(16, self._tick)
```

**单点音符（Tap）绘制：**

```python
def _draw_tap(self, x1, x2, y, color):
    # 主体矩形
    self._canvas.create_rectangle(
        x1 + 4, y - 10, x2 - 4, y + 10,
        fill=color, outline="white", width=1, tags="note"
    )
    # 高光条（顶部 3px 白色）
    self._canvas.create_rectangle(
        x1 + 4, y - 10, x2 - 4, y - 7,
        fill="white", outline="", tags="note"
    )
```

**长按音符（Hold）绘制：**

```python
def _draw_hold(self, x1, x2, y_bottom, hold_h, color):
    y_top = y_bottom - hold_h
    # 主体（圆角用多边形模拟）
    self._canvas.create_rectangle(
        x1 + 8, y_top, x2 - 8, y_bottom,
        fill=color, outline=color, width=0, tags="note"
    )
    # 顶部 cap（亮色）
    self._canvas.create_rectangle(
        x1 + 4, y_top - 8, x2 - 4, y_top + 8,
        fill="white", outline="white", width=1, tags="note"
    )
    # 底部 cap（判定端）
    self._canvas.create_rectangle(
        x1 + 4, y_bottom - 8, x2 - 4, y_bottom + 8,
        fill=color, outline="white", width=1, tags="note"
    )
```

### 5.3 静态背景层（一次性绘制）

背景只在 Canvas 初始化或 resize 时重绘，不参与每帧刷新：

```python
def _draw_background(self):
    w, h = self._canvas.winfo_width(), self._canvas.winfo_height()

    # 轨道背景色交替
    for i in range(4):
        x1 = i * (w // 4)
        x2 = (i + 1) * (w // 4)
        fill = "#1e1e2e" if i % 2 == 0 else "#252535"
        self._canvas.create_rectangle(x1, 0, x2, h, fill=fill, outline="", tags="bg")

    # 轨道分隔线
    for i in range(1, 4):
        x = i * (w // 4)
        self._canvas.create_line(x, 0, x, h, fill="#444466", width=1, tags="bg")

    # 判定线（双线 + 光晕感）
    jy = h * 0.85
    self._canvas.create_line(0, jy + 2, w, jy + 2, fill="#555577", width=3, tags="bg")
    self._canvas.create_line(0, jy,     w, jy,     fill="#ffffff", width=2, tags="bg")

    # 判定区域色块（底部固定）
    for i, color in enumerate(LANE_COLORS):
        x1 = i * (w // 4) + 4
        x2 = (i + 1) * (w // 4) - 4
        self._canvas.create_rectangle(
            x1, jy + 4, x2, h - 8,
            fill=_darken(color), outline=color, width=1, tags="bg"
        )
```

## 7. 关键技术难点与解决方案

### 7.1 进度条拖动 vs 轮询更新的冲突

**问题：** `_poll()` 每 100ms 更新 Slider 位置，用户拖动时会被强制重置。

**解决：** 用 `_user_dragging` 标志位，`<ButtonPress-1>` 时暂停轮询写入，`<ButtonRelease-1>` 时 seek 再恢复。

### 7.2 pygame.mixer 在非主线程初始化的问题

**问题：** tkinter 和 pygame 均有 GUI 事件循环，混用时 pygame 的 `init()` 需在主线程调用。

**解决：** AudioPlayer 在 App `__init__` 中实例化（主线程），播放操作均通过 `player.play()` 等方法调用，pygame.mixer 本身的播放是异步的，不阻塞 tkinter 主循环。

### 7.3 Canvas 静态背景 + 动态音符的分层刷新

**问题：** 每帧 `delete("all")` 再重绘背景性能浪费，会造成闪烁。

**解决：** 背景层打 tag `"bg"`，音符层打 tag `"note"`。每帧只 `delete("note")`，背景永久保留。背景仅在窗口 resize 时重绘。

### 7.4 谱面未生成时预览的处理

**问题：** 用户可能在生成谱面前就点击播放。

**解决：** 未生成谱面时，预览 Canvas 只显示静态背景 + "暂无谱面，请先生成"的提示文字，不启动 `_tick` 循环。生成完成后调用 `canvas.load_chart(chart)` 触发预览就绪。

### 7.5 音频结束自动停止

**问题：** pygame 播放结束后不自动更新 UI 状态。

**解决：** `_poll()` 中检测 `get_position() >= duration`，自动调用 `stop()` 并重置按钮状态：

```python
if self._player.state == "PLAYING":
    pos = self._player.get_position()
    if pos >= self._player.duration:
        self._player.stop()
        self._on_playback_ended()
```
